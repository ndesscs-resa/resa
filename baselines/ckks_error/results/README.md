# Baseline Result Files

This directory keeps the result files that are part of the public artifact.

## Files

| File | Role |
|------|------|
| `recall_lattigo.json` | Small synthetic smoke run generated by `make recall-benchmark` |
| `recall_fullscale_nq_v3.json` | Full-corpus plaintext ranking plus HE candidate-set validation |
| `recall_fullscale_msmarco-distilbert_v3.json` | Full-corpus plaintext ranking plus HE candidate-set validation |
| `recall_fullscale_beir-cohere_v3.json` | Full-corpus plaintext ranking plus HE candidate-set validation |

## Interpretation

The full-scale files report exact plaintext top-k ranking over the full corpus.
The bundled v3 files use `sample` validation mode over 1,000 corpus vectors per
query. The `recall_fullscale` command also supports
`--he-mode boundary`, which validates the full-corpus top-k boundary candidates
selected by the plaintext scan. The result files therefore separate full-corpus
ranking from the HE candidate set used for arithmetic validation. The
submission metric is `paper_mrr@10 = he_sample_mrr@10`; Recall@1 and
Recall@10 are retained as auxiliary fields.

## Validation

Run from the repository root:

```bash
make -C baselines/ckks_error verify-results
```

The validator fails if stale or non-evidence result files are present, if SIFT10M
is presented as a text-retrieval workload, if a full-scale result is missing
reproducibility metadata, or if required HE fidelity fields are absent. It also
fails if the submission metric is not `MRR@10`.
